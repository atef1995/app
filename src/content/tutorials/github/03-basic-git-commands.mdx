## Setting Up Your Git Environment

Before we dive into commands, let's make sure your Git is ready for action.

### Installing Git

**Windows:** Download from [git-scm.com](https://git-scm.com/) and install
**Mac:** Install Xcode Command Line Tools: `xcode-select --install`
**Linux:** `sudo apt install git` (Ubuntu) or equivalent for your distro

### Your Git Identity

Every commit you make needs to know who you are. Let's set that up:

```bash
git config --global user.name "Your Actual Name"
git config --global user.email "your.email@example.com"
```

**Important:** Use the same email you registered with GitHub! This connects your commits to your GitHub profile.

**Pro tip:** Check your settings with:
```bash
git config --list
```

## The Essential Commands: Your Daily Toolkit

Let's learn the commands you'll use every single day, in the order you'll typically use them.

### 1. `git clone` - Getting Projects onto Your Computer

This is like "downloading" a repository from GitHub to your computer, but way cooler because it includes the entire history and sets up everything for development.

**Basic syntax:**
```bash
git clone https://github.com/username/repository-name.git
```

**Real example:**
```bash
git clone https://github.com/your-username/your-username.github.io.git
```

**What actually happens when you clone:**
1. Downloads all project files
2. Downloads the entire Git history
3. Sets up a connection to the GitHub repository
4. Creates a local folder with the project name

**Pro tips:**
- Clone into a dedicated folder like `~/Projects/` or `~/Code/`
- The repository name becomes the folder name automatically
- You can rename the folder: `git clone <url> my-custom-name`

**Navigation after cloning:**
```bash
cd repository-name  # Enter the project folder
ls -la              # See all files (including hidden .git folder)
```

### 2. `git status` - Your Git Compass

This is your best friend. It tells you exactly what's going on in your repository at any moment.

```bash
git status
```

**What you'll see:**
- Which files have been changed
- Which changes are staged for commit
- Which files aren't being tracked
- What branch you're on
- Whether you're ahead/behind the remote repository

**Pro tip:** Run `git status` constantly! Before commits, after changes, when you're confused - it's like asking "where am I and what's happening?"

### 3. `git add` - Staging Your Changes

Think of this as selecting which changes you want to include in your next "save point" (commit).

**Add a specific file:**
```bash
git add filename.txt
```

**Add multiple files:**
```bash
git add file1.txt file2.css file3.js
```

**Add all changed files:**
```bash
git add .
```

**The staging concept explained:**
Imagine you're packing a suitcase (commit). `git add` puts items into your packing area, but doesn't close the suitcase yet. This lets you:
- Review what you're about to commit
- Add more changes if needed
- Remove things from staging if you change your mind

**Unstaging if you change your mind:**
```bash
git reset filename.txt    # Remove specific file from staging
git reset                 # Remove all files from staging
```

### 4. `git commit` - Creating Save Points

This is where the magic happens - you're creating a permanent snapshot of your project at this moment.

**Basic commit:**
```bash
git commit -m "Your commit message here"
```

**What makes a great commit message:**

**‚úÖ Good examples:**
- `Add user login functionality`
- `Fix mobile navigation bug`
- `Update README with installation instructions`
- `Remove unused CSS styles`

**‚ùå Bad examples:**
- `changes`
- `update stuff`
- `fix`
- `asdfkj`

**The commit message formula:**
`[Action] [What] [Why if needed]`
- Action: Add, Fix, Update, Remove, Refactor
- What: Be specific about what you changed
- Why: Only if it's not obvious

**Advanced commit with longer description:**
```bash
git commit -m "Add user authentication system

- Implement login and logout functionality
- Add password hashing with bcrypt
- Create user session management
- Update database schema for user table"
```

### 5. `git push` - Sending Changes to GitHub

This uploads your commits to GitHub so others can see them and you have a backup.

**First time pushing a new repository:**
```bash
git push -u origin main
```

**Regular pushes after that:**
```bash
git push
```

**What's happening:**
- Your commits are uploaded to GitHub
- Your GitHub repository is updated
- Other people can now see your changes
- Your contribution graph gets those beautiful green squares!

**Troubleshooting push issues:**
If push fails, it's usually because someone else made changes. Fix it with:
```bash
git pull    # Download and merge changes
git push    # Try pushing again
```

### 6. `git pull` - Getting Latest Changes

This downloads any new commits from GitHub and integrates them into your local repository.

```bash
git pull
```

**When to use it:**
- Before starting new work
- When collaborating with others
- After someone else pushes changes
- As part of your daily routine

**What actually happens:**
1. Downloads new commits from GitHub
2. Automatically merges them with your local changes
3. Updates your working files

**Best practice:** Always pull before pushing, especially when working with others.

### 7. `git log` - Viewing Project History

See the history of commits in your project.

**Basic log:**
```bash
git log
```

**Pretty, concise log:**
```bash
git log --oneline
```

**Log with graph visualization:**
```bash
git log --oneline --graph
```

**What you'll see:**
- Commit hashes (unique IDs)
- Author names and emails
- Commit dates
- Commit messages

**Navigation in log:**
- Press `q` to exit
- Use arrow keys to scroll
- Press `/` to search

## A Real-World Workflow Example

Let's put it all together with a realistic scenario: You want to update your portfolio website.

**1. Start your development session:**
```bash
cd ~/Projects/my-portfolio    # Navigate to your project
git status                    # Check current state
git pull                      # Get any remote changes
```

**2. Make your changes:**
- Edit your HTML, CSS, or other files
- Add new projects, update skills, etc.

**3. Review and stage your changes:**
```bash
git status                    # See what you've changed
git add .                     # Stage all changes
git status                    # Confirm what's staged
```

**4. Commit your changes:**
```bash
git commit -m "Update portfolio with new JavaScript project"
```

**5. Share with the world:**
```bash
git push
```

**That's it!** Your changes are now live on GitHub, and if you're using GitHub Pages, your website is automatically updated too!

## Advanced Commands for Everyday Use

Once you're comfortable with the basics, these commands will make your life even easier:

### `git diff` - See What Changed

**See unstaged changes:**
```bash
git diff
```

**See staged changes:**
```bash
git diff --cached
```

**Compare with specific commit:**
```bash
git diff HEAD~1    # Compare with previous commit
```

### `git checkout` - Time Travel and Branch Switching

**Create and switch to a new branch:**
```bash
git checkout -b new-feature
```

**Switch to existing branch:**
```bash
git checkout main
```

**Undo changes to a file:**
```bash
git checkout -- filename.txt
```

### `git branch` - Branch Management

**List all branches:**
```bash
git branch
```

**Create new branch:**
```bash
git branch feature-name
```

**Delete branch:**
```bash
git branch -d feature-name
```

## Common Scenarios and Solutions

### "Help! I Made a Mistake!"

**Scenario 1: Bad commit message**
```bash
git commit --amend -m "Corrected commit message"
```

**Scenario 2: Forgot to add a file**
```bash
git add forgotten-file.txt
git commit --amend --no-edit
```

**Scenario 3: Want to undo last commit (but keep changes)**
```bash
git reset HEAD~1
```

**Scenario 4: Want to completely undo last commit**
```bash
git reset --hard HEAD~1
```

### "I'm Confused About What's Happening!"

**Your diagnostic toolkit:**
```bash
git status          # What's the current state?
git log --oneline   # What commits exist?
git remote -v       # Where am I connected?
git branch          # What branch am I on?
```

### "Someone Else Changed the Same File!"

This is called a merge conflict. Don't panic!

**When it happens:**
```bash
git pull
# Auto-merging filename.txt
# CONFLICT (content): Merge conflict in filename.txt
```

**How to fix it:**
1. Open the conflicted file
2. Look for conflict markers: `<<<<<<<`, `=======`, `>>>>>>>`
3. Decide which changes to keep
4. Remove the conflict markers
5. Add and commit the resolved file

```bash
git add filename.txt
git commit -m "Resolve merge conflict in filename.txt"
```

## Troubleshooting: When Things Go Wrong

### "Command not found"
Git isn't installed or not in your PATH. Reinstall Git and restart your terminal.

### "Permission denied (publickey)"
Your SSH keys aren't set up. Use HTTPS URLs for now, or set up SSH keys following GitHub's guide.

### "Your branch is behind"
Someone else pushed changes. Run `git pull` before pushing.

### "Repository not found"
- Check the URL is correct
- Make sure you have access to the repository
- For private repos, ensure you're logged in

### "Merge conflict"
Don't panic! Follow the conflict resolution steps above.

### "Detached HEAD"
You've checked out a specific commit instead of a branch. Create a new branch:
```bash
git checkout -b rescue-branch
```
