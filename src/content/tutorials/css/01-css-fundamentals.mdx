## What is CSS?

CSS stands for **Cascading Style Sheets**. It's a stylesheet language used to describe the presentation of HTML documents. While HTML provides the structure, CSS provides the styling.

## Basic CSS Syntax

CSS follows a simple syntax pattern:

```css
selector {
  property: value;
  property: value;
}
```

Let's break this down:
- **Selector**: Targets the HTML element(s) you want to style
- **Property**: The style attribute you want to change (e.g., color, font-size)
- **Value**: The specific setting for that property

## CSS Selectors

CSS selectors are like targeting systems that tell your styles exactly which elements to affect. Let's explore the three main types with hands-on practice!

### Element Selector
Targets all elements of a specific type:

```css
p {
  color: blue;
  font-size: 16px;
}
```

### Class Selector
Targets elements with a specific class attribute:

```css
.highlight {
  background-color: yellow;
  font-weight: bold;
}
```

### ID Selector
Targets an element with a specific ID:

```css
#header {
  background-color: navy;
  color: white;
  padding: 20px;
}
```

## Common CSS Properties

### Colors
```css
.text-styles {
  color: red;           /* Text color */
  background-color: #f0f0f0;  /* Background color */
}
```

### Typography
```css
.typography {
  font-family: Arial, sans-serif;
  font-size: 18px;
  font-weight: bold;
  text-align: center;
  line-height: 1.5;
}
```

### Spacing
```css
.spacing {
  margin: 20px;         /* Space outside the element */
  padding: 15px;        /* Space inside the element */
}
```

### Box Model
Every HTML element is essentially a rectangular box. The CSS box model consists of:
- **Content**: The actual content of the element
- **Padding**: Space between content and border
- **Border**: A line around the padding
- **Margin**: Space outside the border

## CSS Units

### Absolute Units
- **px**: Pixels (most common)
- **pt**: Points
- **cm**: Centimeters

### Relative Units
- **%**: Percentage of parent element
- **em**: Relative to current font size
- **rem**: Relative to root font size
- **vh/vw**: Viewport height/width

```css
.units-example {
  width: 50%;           /* 50% of parent width */
  font-size: 1.5rem;    /* 1.5 times the root font size */
  margin: 2em;          /* 2 times the current font size */
  height: 50vh;         /* 50% of viewport height */
}
```

## Display Property

Controls how an element is displayed:

```css
.block {
  display: block;       /* Takes full width, stacks vertically */
}

.inline {
  display: inline;      /* Takes only needed width, flows horizontally */
}

.inline-block {
  display: inline-block; /* Hybrid of inline and block */
}

.none {
  display: none;        /* Element is hidden */
}
```

## CSS Cascade and Specificity

CSS follows these rules when multiple styles conflict:

1. **Inline styles** (highest priority)
2. **IDs** (#header)
3. **Classes** (.highlight)
4. **Elements** (p, div)

```css
/* This will be overridden */
p {
  color: blue;
}

/* This has higher specificity */
.special-text {
  color: red;
}

/* This has even higher specificity */
#unique-paragraph {
  color: green;
}
```

## Best Practices

1. **Use external stylesheets** for better maintainability
2. **Use classes** instead of IDs for styling (IDs should be unique)
3. **Keep specificity low** to make styles easier to override
4. **Use meaningful class names** (.nav-button instead of .red-button)
5. **Group related styles** together
6. **Comment your CSS** for complex sections

```css
/* Navigation Styles */
.nav {
  background-color: #333;
  padding: 1rem;
}

.nav-link {
  color: white;
  text-decoration: none;
  margin-right: 1rem;
}

.nav-link:hover {
  color: #f39c12;
}
```
