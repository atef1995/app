# Advanced CSS Selectors

Welcome to Advanced CSS Selectors! In this tutorial, you'll learn powerful techniques to target specific elements with precision, create dynamic styles, and build more maintainable CSS.

## Understanding CSS Selector Specificity

Before diving into advanced selectors, it's crucial to understand how CSS calculates specificity:

- **Inline styles**: 1000 points
- **IDs**: 100 points
- **Classes, attributes, pseudo-classes**: 10 points
- **Elements and pseudo-elements**: 1 point

<SeparatedEditorPreview 
  title="CSS Specificity Demo"
  mission="Understand how CSS specificity affects styling"
  instructions="See which styles take precedence based on selector specificity"
initialHtml={`
<div class="specificity-demo">
  <p id="specific-paragraph" class="blue-text">
    This paragraph demonstrates CSS specificity rules.
  </p>
  <p class="blue-text special-text">
    This paragraph shows class combination effects.
  </p>
  <p class="blue-text">
    This paragraph uses only one class.
  </p>
</div>
`}

initialCss={`
/* Specificity: 1 (element) */
p {
  color: black;
  font-size: 16px;
  padding: 15px;
  background: #f8f9fa;
  margin: 10px 0;
  border-radius: 6px;
}

/* Specificity: 10 (class) */
.blue-text {
  color: #3498db;
  font-weight: bold;
}

/* Specificity: 20 (two classes) */
.blue-text.special-text {
  color: #e74c3c;
  background: #fff5f5;
  border: 2px solid #e74c3c;
}

/* Specificity: 110 (ID + class) */
#specific-paragraph.blue-text {
  color: #27ae60;
  background: #f0fff4;
  border: 2px solid #27ae60;
  font-style: italic;
}

.specificity-demo {
  padding: 20px;
  background: #ecf0f1;
  border-radius: 8px;
  font-family: Arial, sans-serif;
}
`}/>

📚 **How This Demo Works:**
This example demonstrates CSS specificity hierarchy and how the browser determines which styles to apply when multiple selectors target the same element:

- **Element selector `p`** (specificity: 1) - Base styling applied to all paragraphs
- **Class selector `.blue-text`** (specificity: 10) - Overrides element styles, makes text blue  
- **Two classes `.blue-text.special-text`** (specificity: 20) - Higher specificity, makes text red
- **ID + class `#specific-paragraph.blue-text`** (specificity: 110) - Highest specificity, makes text green

The **specificity calculation** adds up points: IDs (100) + Classes (10) + Elements (1). Higher totals win!

## Pseudo-Classes

Pseudo-classes select elements based on their state or position:

### Structural Pseudo-Classes

initialCss={`
.structural-demo {
  padding: 20px;
  background: #2c3e50;
  color: white;
  font-family: Arial, sans-serif;
}

.product-list {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 15px;
  margin-top: 20px;
}

.product {
  padding: 20px;
  background: #34495e;
  border-radius: 8px;
  text-align: center;
  font-weight: bold;
  transition: all 0.3s ease;
}

/* First child gets special styling */
.product:first-child {
  background: #e74c3c;
  transform: scale(1.05);
}

/* Last child gets different styling */
.product:last-child {
  background: #27ae60;
  transform: scale(1.05);
}

/* Every odd child (1st, 3rd, 5th, etc.) */
.product:nth-child(odd) {
  border-left: 4px solid #3498db;
}

/* Every even child (2nd, 4th, 6th, etc.) */
.product:nth-child(even) {
  border-left: 4px solid #f39c12;
}

/* Every third child (3rd, 6th, 9th, etc.) */
.product:nth-child(3n) {
  background: #9b59b6;
  color: white;
}

/* Second child specifically */
.product:nth-child(2) {
  background: #1abc9c;
}

/* Only child (if there's only one) */
.product:only-child {
  background: #e67e22;
  grid-column: 1 / -1;
  font-size: 18px;
}
`}/>

📚 **How This Demo Works:**
This demo showcases powerful structural pseudo-classes that select elements based on their position within the DOM tree:

- **`:first-child`** - Targets the first product (red background with scale transform)
- **`:last-child`** - Targets the last product (green background with scale transform)  
- **`:nth-child(odd)`** - Every 1st, 3rd, 5th element gets blue left border
- **`:nth-child(even)`** - Every 2nd, 4th, 6th element gets orange left border
- **`:nth-child(3n)`** - Every 3rd element (3, 6, 9...) gets purple background
- **`:nth-child(2)`** - Specifically targets the 2nd element with teal background

The **CSS Grid** layout automatically adjusts columns based on available space using `repeat(auto-fit, minmax(200px, 1fr))`.

## Performance and Best Practices

### Selector Performance Tips

1. **Avoid Universal Selectors**: `* {}` can be slow on large DOMs
2. **Be Specific**: Use classes instead of complex descendant selectors
3. **Minimize Nesting**: Deep nesting impacts performance
4. **Use Modern Selectors**: `:is()` and `:where()` are more performant

```css
/* Slower */
div > ul > li > a:hover {}

/* Faster */
.nav-link:hover {}
```

### Maintainable Selector Strategies

1. **Use BEM Methodology**: Block, Element, Modifier naming
2. **Avoid !important**: Use specificity properly instead
3. **Group Related Selectors**: Keep similar styles together
4. **Document Complex Selectors**: Add comments for complex logic

## Modern CSS Selector Features

### :is() and :where() Pseudo-Classes

```css
/* Instead of writing */
h1, h2, h3, h4, h5, h6 {
  font-family: sans-serif;
}

/* You can write */
:is(h1, h2, h3, h4, h5, h6) {
  font-family: sans-serif;
}

/* :where() has 0 specificity */
:where(.btn, .link, .action) {
  padding: 8px 16px;
  border-radius: 4px;
}
```

### :has() Relational Pseudo-Class

```css
/* Style cards that contain images */
.card:has(img) {
  padding: 0;
}

/* Style form sections with errors */
.form-section:has(.error) {
  border-color: red;
}
```
