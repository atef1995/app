# Control Structures: Making Decisions in JavaScript

Welcome to the world of **control structures**! 🎛️ This is where your programs come alive and start making intelligent decisions. Control structures are the building blocks that allow your code to respond differently based on conditions and repeat actions efficiently.

## What are Control Structures?

**Control structures** determine the flow of execution in your program. Instead of code running line by line from top to bottom, control structures allow you to:

- **Make decisions** (if/else statements)
- **Repeat actions** (loops)
- **Choose between multiple options** (switch statements)
- **Jump out of or skip iterations** (break/continue)

Think of control structures as the **traffic lights and road signs** of your code - they direct the flow of execution based on conditions.

## Conditional Statements

Conditional statements allow your program to make decisions and execute different code paths based on **boolean expressions** (conditions that evaluate to true or false).

### The `if` Statement

The most fundamental control structure. It executes code only when a condition is true.

}`}
</InteractiveCodeBlock>

**Syntax breakdown:**
- `if` keyword
- Condition in parentheses `()`
- Code block in curly braces `{}`

### The `if...else` Statement

When you need to execute different code for true and false conditions.

<InteractiveCodeBlock title="if...else Decisions" description="Different paths for different conditions:">
{`let age = 17;

if (age >= 18) {
    console.log("You can vote! 🗳️");
} else {
    console.log("You'll be able to vote in", 18 - age, "years");
}

// Weather example
let weather = "sunny";

if (weather === "rainy") {
    console.log("Take an umbrella! ☔");
} else {
    console.log("Enjoy the nice weather! 🌈");
}`}
</InteractiveCodeBlock>

> **💡 Comparison Operators** Use `===` for **strict equality** (checks both value and type). Use `==` for **loose equality** (converts types before comparing). It's recommended to use `===` to avoid unexpected behavior.

### Logical Operators

Combine multiple conditions using **logical operators** for more complex decision-making.

<InteractiveCodeBlock title="Logical Operators" description="Combine conditions with AND, OR, and NOT:">
{`let age = 25;
let hasLicense = true;
let hasInsurance = true;

// AND operator (&&) - ALL conditions must be true
if (age >= 18 && hasLicense && hasInsurance) {
    console.log("You can drive! 🚗");
} else {
    console.log("You cannot drive yet");
}

// OR operator (||) - AT LEAST ONE condition must be true
let isWeekend = false;
let isHoliday = true;

if (isWeekend || isHoliday) {
    console.log("No work today! 🎉");
} else {
    console.log("Time to work! 💼");
}

// NOT operator (!) - Reverses the boolean value
let isLoggedIn = false;

if (!isLoggedIn) {
    console.log("Please log in first 🔐");
} else {
    console.log("Welcome back! 👋");
}`}
</InteractiveCodeBlock>

> **💡 Short-Circuit Evaluation** JavaScript uses **short-circuit evaluation** with logical operators:
> - `&&`: If the first condition is false, the second isn't even checked
> - `||`: If the first condition is true, the second isn't checked
> 
> This can be used for **conditional execution**:
> ```javascript
> user && console.log("User exists!"); // Only runs if user is truthy
> user || console.log("No user found"); // Only runs if user is falsy
> ```

### The Ternary Operator

A **shorthand** for simple if/else statements. Perfect for **conditional assignment**.

<InteractiveCodeBlock title="Ternary Operator" description="Shorthand conditional expressions:">
{`let score = 85;

// Traditional if/else
let result;
if (score >= 60) {
    result = "Pass";
} else {
    result = "Fail";
}
console.log("Traditional:", result);

// Ternary operator (condition ? trueValue : falseValue)
let quickResult = score >= 60 ? "Pass" : "Fail";
console.log("Ternary:", quickResult);

// More examples
let temperature = 15;
let clothing = temperature > 20 ? "T-shirt 👕" : "Jacket 🧥";
console.log("Wear:", clothing);

let time = 14;
let greeting = time < 12 ? "Good morning! 🌅" : 
               time < 18 ? "Good afternoon! ☀️" : 
               "Good evening! 🌙";
console.log(greeting);`}
</InteractiveCodeBlock>

**Syntax:** `condition ? valueIfTrue : valueIfFalse`

### The `switch` Statement

When you have **multiple discrete values** to check against a single variable. More readable than many `else if` statements for **enumerated values**.

<InteractiveCodeBlock title="Switch Statement" description="Handle multiple specific values:">
{`let day = "wednesday";

switch (day) {
    case "monday":
        console.log("Start of the work week 💼");
        break;
    case "tuesday":
        console.log("Tuesday blues 😴");
        break;
    case "wednesday":
        console.log("Hump day! 🐪");
        break;
    case "thursday":
        console.log("Almost there! 📈");
        break;
    case "friday":
        console.log("TGIF! 🎉");
        break;
    case "saturday":
    case "sunday":  // Fall-through for multiple cases
        console.log("Weekend time! 🏖️");
        break;
    default:
        console.log("That's not a valid day! 🤔");
}

// Another example: Calculator
let operation = "*";
let a = 10;
let b = 5;

switch (operation) {
    case "+":
        console.log(a + b);
        break;
    case "-":
        console.log(a - b);
        break;
    case "*":
        console.log(a * b);
        break;
    case "/":
        console.log(a / b);
        break;
    default:
        console.log("Unknown operation");
}`}
</InteractiveCodeBlock>

> **💡 The `break` Statement** Without `break`, execution **falls through** to the next case. This is sometimes intentional (like weekend days above) but usually you want `break` to exit the switch after matching a case.

## Loops

**Loops** allow you to repeat code multiple times, making your programs efficient and powerful. Instead of writing the same code 100 times, write it once and loop 100 times!

### The `for` Loop

The most common loop, perfect when you know **how many times** you want to repeat something.

<InteractiveCodeBlock title="For Loop Basics" description="Repeat code a specific number of times:">
{`// Basic for loop: for (initialization; condition; increment)
for (let i = 0; i < 5; i++) {
    console.log("Loop iteration:", i);
}

// Counting down
for (let countdown = 5; countdown > 0; countdown--) {
    console.log("Countdown:", countdown);
}
console.log("Blast off! 🚀");

// Working with arrays
let fruits = ["apple", "banana", "cherry", "date"];

for (let i = 0; i < fruits.length; i++) {
    console.log(\`\${i + 1}. \${fruits[i]} 🍎\`);
}`}
</InteractiveCodeBlock>

**For loop components:**
1. **Initialization** (`let i = 0`): Run once at the start
2. **Condition** (`i < 5`): Checked before each iteration
3. **Increment** (`i++`): Run after each iteration

### The `while` Loop

Repeats code **while a condition is true**. Great when you don't know exactly how many iterations you need.

<InteractiveCodeBlock title="While Loop" description="Repeat while a condition remains true:">
{`// Basic while loop
let count = 0;

while (count < 3) {
    console.log("Count is:", count);
    count++; // Don't forget to increment!
}

// Practical example: Finding a number
let target = 7;
let guess = 1;

while (guess !== target) {
    console.log("Guessing:", guess);
    guess++;
}
console.log("Found it! The number was", target);

// Password attempts
let attempts = 0;
let maxAttempts = 3;
let correctPassword = "secret123";
let userPassword = "wrong";

while (userPassword !== correctPassword && attempts < maxAttempts) {
    attempts++;
    console.log(\`Attempt \${attempts}: Wrong password!\`);
    // In real code, you'd get input from user here
    if (attempts === 2) userPassword = "secret123"; // Simulate correct guess
}

if (userPassword === correctPassword) {
    console.log("Access granted! 🔓");
} else {
    console.log("Too many failed attempts! 🔒");
}`}
</InteractiveCodeBlock>

> **⚠️ Infinite Loops** Be careful with while loops! If the condition never becomes false, you'll create an **infinite loop** that can crash your program. Always make sure the loop variable changes inside the loop.

### The `do...while` Loop

Similar to `while`, but **always runs at least once** because the condition is checked after the loop body.

<InteractiveCodeBlock title="Do-While Loop" description="Execute at least once, then check condition:">
{`// Regular while loop - might not run at all
let number = 10;
while (number < 5) {
    console.log("While loop:", number);
    number++;
}
console.log("While loop didn't run because 10 is not < 5");

// Do-while loop - always runs at least once
let anotherNumber = 10;
do {
    console.log("Do-while loop:", anotherNumber);
    anotherNumber++;
} while (anotherNumber < 5);
console.log("Do-while ran once even though 10 is not < 5");

// Practical example: Menu system
let choice;
do {
    console.log("\\n=== Menu ===");
    console.log("1. Play Game");
    console.log("2. Settings");
    console.log("3. Exit");
    
    // Simulate user input
    choice = Math.floor(Math.random() * 4) + 1; // Random choice
    console.log("User chose:", choice);
    
    switch (choice) {
        case 1:
            console.log("Starting game... 🎮");
            break;
        case 2:
            console.log("Opening settings... ⚙️");
            break;
        case 3:
            console.log("Goodbye! 👋");
            break;
        default:
            console.log("Invalid choice, try again!");
    }
} while (choice !== 3);`}
</InteractiveCodeBlock>

### The `for...of` Loop

**Modern loop** for iterating over **iterable objects** like arrays, strings, and more. Much cleaner than traditional for loops when you need the values, not the indexes.

<InteractiveCodeBlock title="For...of Loop" description="Iterate over values in collections:">
{`// Array iteration
let animals = ["cat", "dog", "bird", "fish"];

// Traditional for loop
console.log("Traditional for loop:");
for (let i = 0; i < animals.length; i++) {
    console.log(animals[i]);
}

// for...of loop (cleaner!)
console.log("\\nFor...of loop:");
for (let animal of animals) {
    console.log(animal, "🐾");
}

// String iteration
let word = "JavaScript";
console.log("\\nLetters in", word + ":");
for (let letter of word) {
    console.log(letter);
}

// With index using entries()
console.log("\\nWith indexes:");
for (let [index, animal] of animals.entries()) {
    console.log(\`\${index}: \${animal}\`);
}`}
</InteractiveCodeBlock>

### The `for...in` Loop

Iterates over **enumerable properties** of objects. Use this for objects, not arrays.

<InteractiveCodeBlock title="For...in Loop" description="Iterate over object properties:">
{`// Object iteration
let person = {
    name: "Alice",
    age: 30,
    city: "New York",
    job: "Developer"
};

console.log("Person details:");
for (let property in person) {
    console.log(\`\${property}: \${person[property]}\`);
}

// Counting properties
let propertyCount = 0;
for (let prop in person) {
    propertyCount++;
}
console.log("\\nTotal properties:", propertyCount);

// Checking for specific properties
let requiredFields = ["name", "age"];
let missingFields = [];

for (let field of requiredFields) {
    if (!(field in person)) {
        missingFields.push(field);
    }
}

if (missingFields.length === 0) {
    console.log("All required fields present! ✅");
} else {
    console.log("Missing fields:", missingFields);
}`}
</InteractiveCodeBlock>

> **💡 for...of vs for...in**
> - `for...of`: Gets **values** from iterables (arrays, strings, etc.)
> - `for...in`: Gets **property names** from objects
> 
> Remember: "of" for **o**bject values, "in" for property **n**ames (sort of!)

## Loop Control: `break` and `continue`

Control the flow within loops using **jump statements**.

### The `break` Statement

**Exits the loop immediately**, even if the loop condition is still true.

<InteractiveCodeBlock title="Break Statement" description="Exit loops early when needed:">
{`// Finding first even number
let numbers = [1, 3, 7, 8, 12, 15];
let firstEven;

for (let num of numbers) {
    console.log("Checking:", num);
    if (num % 2 === 0) {
        firstEven = num;
        console.log("Found first even number:", firstEven);
        break; // Exit the loop immediately
    }
}

// Search example
let inventory = ["apples", "bananas", "grapes", "oranges"];
let searchItem = "grapes";
let found = false;

for (let i = 0; i < inventory.length; i++) {
    console.log(\`Searching position \${i}: \${inventory[i]}\`);
    if (inventory[i] === searchItem) {
        console.log(\`Found \${searchItem} at position \${i}! 🎯\`);
        found = true;
        break; // No need to continue searching
    }
}

if (!found) {
    console.log(searchItem, "not found in inventory");
}`}
</InteractiveCodeBlock>

### The `continue` Statement

**Skips the rest of the current iteration** and moves to the next one.

<InteractiveCodeBlock title="Continue Statement" description="Skip certain iterations:">
{`// Skip odd numbers
console.log("Even numbers from 1 to 10:");
for (let i = 1; i <= 10; i++) {
    if (i % 2 !== 0) {
        continue; // Skip odd numbers
    }
    console.log(i, "is even");
}

// Processing valid data only
let userInputs = ["", "Alice", null, "Bob", "Charlie", undefined, "Diana"];

console.log("\\nValid names:");
for (let input of userInputs) {
    if (!input || input.length === 0) {
        continue; // Skip invalid entries
    }
    console.log("Processing:", input);
}

// Skip weekends in date processing
let days = ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday"];

console.log("\\nWork days:");
for (let day of days) {
    if (day === "Saturday" || day === "Sunday") {
        continue; // Skip weekends
    }
    console.log(day, "- work day 💼");
}`}
</InteractiveCodeBlock>

## Best Practices

### 1. Choose the Right Loop

Different loops for different scenarios:

```javascript
// ✅ Use for...of for array values
for (let item of array) { /* process item */ }

// ✅ Use for...in for object properties  
for (let key in object) { /* process key */ }

// ✅ Use traditional for when you need the index
for (let i = 0; i < array.length; i++) { /* need i */ }

// ✅ Use while when condition-dependent
while (condition) { /* condition might change */ }
```

### 2. Avoid Deep Nesting

**Deeply nested code** is hard to read and maintain. Use **early returns** and **guard clauses**.

```javascript
// ❌ Deep nesting
if (user) {
    if (user.isActive) {
        if (user.permissions.includes('write')) {
            // do something
        }
    }
}

// ✅ Early returns / Guard clauses
if (!user) return;
if (!user.isActive) return;
if (!user.permissions.includes('write')) return;
// do something
```

### 3. Use Descriptive Variable Names

Loop variables should be meaningful, especially in nested loops.

```javascript
// ❌ Unclear
for (let i = 0; i < users.length; i++) {
    for (let j = 0; j < posts.length; j++) {
        // What are i and j?
    }
}

// ✅ Clear
for (let userIndex = 0; userIndex < users.length; userIndex++) {
    for (let postIndex = 0; postIndex < posts.length; postIndex++) {
        // Much clearer!
    }
}
```

### 4. Prevent Infinite Loops

Always ensure loop conditions will eventually become false.

```javascript
// ❌ Potential infinite loop
let count = 0;
while (count < 10) {
    console.log(count);
    // Forgot to increment count!
}

// ✅ Proper increment
let count = 0;
while (count < 10) {
    console.log(count);
    count++; // Ensures loop will end
}
```

## Real-World Example: Simple Game Logic

Let's put it all together in a practical example:

<InteractiveCodeBlock title="Number Guessing Game" description="A complete game using all control structures:">
{`// Number Guessing Game
function playGuessingGame() {
    const targetNumber = Math.floor(Math.random() * 10) + 1; // 1-10
    const maxAttempts = 3;
    let attempts = 0;
    let won = false;
    
    console.log("🎯 Welcome to the Number Guessing Game!");
    console.log("I'm thinking of a number between 1 and 10");
    console.log(\`You have \${maxAttempts} attempts.\\n\`);
    
    // Simulate user guesses
    let guesses = [5, 8, targetNumber]; // Mock user input
    
    for (let guess of guesses) {
        attempts++;
        console.log(\`Attempt \${attempts}: You guessed \${guess}\`);
        
        if (guess === targetNumber) {
            won = true;
            console.log("🎉 Congratulations! You guessed it!");
            break; // Exit the game
        } else if (guess < targetNumber) {
            console.log("📈 Too low!");
        } else {
            console.log("📉 Too high!");
        }
        
        // Check if out of attempts
        if (attempts >= maxAttempts) {
            break;
        }
        
        // Show remaining attempts
        let remaining = maxAttempts - attempts;
        if (remaining > 0) {
            console.log(\`\${remaining} attempt(s) remaining\\n\`);
        }
    }
    
    // Game over logic
    if (!won) {
        console.log(\`😞 Game Over! The number was \${targetNumber}\`);
    }
    
    // Final stats
    console.log(\`\\nGame Stats:\`);
    console.log(\`- Target number: \${targetNumber}\`);
    console.log(\`- Attempts used: \${attempts}/\${maxAttempts}\`);
    console.log(\`- Result: \${won ? "Victory! 🏆" : "Defeat 💪 Try again!"}\`);
}

playGuessingGame();`}
</InteractiveCodeBlock>

## Common Patterns and Idioms

### Finding Maximum/Minimum

```javascript
let numbers = [23, 45, 12, 67, 34];
let max = numbers[0];

for (let num of numbers) {
    if (num > max) {
        max = num;
    }
}
console.log("Maximum:", max);
```

### Counting and Filtering

```javascript
let ages = [12, 25, 17, 30, 16, 45];
let adults = 0;

for (let age of ages) {
    if (age >= 18) {
        adults++;
    }
}
console.log("Adults:", adults);
```

### Input Validation Loop

```javascript
let validInput = false;
do {
    // Get user input (simulated)
    let input = Math.floor(Math.random() * 15) + 1;
    console.log("Input:", input);
    
    if (input >= 1 && input <= 10) {
        console.log("Valid input accepted!");
        validInput = true;
    } else {
        console.log("Invalid! Please enter 1-10");
    }
} while (!validInput);
```
