## What are Operators? (The Building Blocks of Logic!)

Think about it like this: **operators are like verbs in English** - they tell JavaScript what ACTION to perform on your data.

- `+` means "add these together" 
- `===` means "are these exactly the same?"
- `&&` means "are BOTH of these true?"

**Real-world moment**: Every time you:
- Like a post (checking if you're logged in `&&` if the post exists)
- Add items to a cart (using `+=` to increase the total)
- Check your age for a website (using `>=` to see if you're old enough)

You're using operators!

## Common Pitfalls to Avoid

### 1. Using == Instead of ===

```javascript
// âŒ Avoid loose equality
console.log(0 == false);   // true (confusing!)
console.log("" == false);  // true (confusing!)
console.log(null == undefined); // true (confusing!)

// âœ… Use strict equality
console.log(0 === false);  // false (clear!)
console.log("" === false); // false (clear!)
console.log(null === undefined); // false (clear!)
```

### 2. Forgetting Operator Precedence

```javascript
// âŒ Confusing without parentheses
let result = 5 + 3 * 2; // 11, not 16!

// âœ… Clear with parentheses
let result1 = 5 + (3 * 2); // 11 (explicit)
let result2 = (5 + 3) * 2; // 16 (different intention)
```

### 3. Assignment in Conditions

```javascript
// âŒ Accidental assignment
let x = 5;
if (x = 10) { // Should be x === 10
  console.log("This always runs!"); // x is now 10
}

// âœ… Proper comparison
let y = 5;
if (y === 10) {
  console.log("This won't run");
}
```
