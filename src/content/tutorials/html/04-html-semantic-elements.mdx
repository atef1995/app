# HTML5 Semantic Elements: Building Meaningful Web Structure

Welcome to semantic HTML! üèóÔ∏è Semantic elements are the difference between a house of cards and a well-engineered building. They provide meaning, structure, and context that benefits users, search engines, and assistive technologies.

## What is Semantic HTML?

Semantic HTML uses elements that clearly describe their meaning and purpose, rather than just their appearance. It's about **what** content is, not **how** it looks.

> **‚ö†Ô∏è Important** There should only be one `<main>` element per page, and it should not be inside `<article>`, `<aside>`, `<footer>`, `<header>`, or `<nav>` elements.

## Benefits of Semantic HTML

### 1. SEO Advantages

Search engines understand content structure better:

- **Better rankings** for well-structured content
- **Rich snippets** in search results
- **Featured snippets** for structured data
- **Improved crawling** and indexing

### 2. Accessibility Benefits

Screen readers and assistive technologies work better:

- **Navigation shortcuts** (jump to main content, navigation)
- **Content structure** understanding
- **Landmark navigation** for efficient browsing
- **Better context** for users with disabilities

### 3. Maintainability

Semantic code is easier to maintain:

- **Clear structure** for other developers
- **Easier debugging** and updates
- **Future-proof** as HTML evolves
- **Better separation** of content and presentation

## Common Semantic HTML Mistakes

### 1. Using Divs for Everything
```html
<!-- ‚ùå Bad -->
<div class="header">
<div class="nav">
<div class="main">
<div class="footer">

<!-- ‚úÖ Good -->
<header>
<nav>
<main>
<footer>
```

### 2. Wrong Element Usage
```html
<!-- ‚ùå Bad -->
<section>Contact Us</section> <!-- Missing heading -->

<!-- ‚úÖ Good -->
<section>
    <h2>Contact Us</h2>
    <p>Get in touch with our team...</p>
</section>
```

### 3. Overusing Article
```html
<!-- ‚ùå Bad -->
<article class="button">Click me</article>

<!-- ‚úÖ Good -->
<button>Click me</button>
```

### 4. Missing Main Element
Every page should have exactly one `<main>` element containing the primary content.
