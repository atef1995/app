# HTML Fundamentals: Building Your First Web Page

Welcome to HTML! üåê HTML (HyperText Markup Language) is the foundation of every web page. It's the skeleton that gives structure to web content.

## What is HTML?

HTML stands for **HyperText Markup Language**. It's not a programming language, but a **markup language** that uses tags to define the structure and content of web pages.

> **üí° Markup vs Programming** HTML is a markup language, not a programming language. It describes content structure rather than logic or behavior. Think of it as the blueprint for a house - it shows where walls, doors, and windows go, but doesn't control electricity or plumbing.

## HTML Document Structure

Every HTML document follows the same basic structure:

### 1. Document Type Declaration

```html
<!DOCTYPE html>
```

The `<!DOCTYPE html>` declaration tells the browser this is an HTML5 document. Always put this at the very top of your HTML files.

### 2. The HTML Root Element

```html
<html lang="en">
```

The `<html>` element is the root of every HTML page. The `lang="en"` attribute tells browsers and screen readers the page language.

### 3. The Head Section

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <meta name="description" content="Page description for search engines">
</head>
```

**Head elements include:**
- `<meta charset="UTF-8">` - Character encoding for international text
- `<meta name="viewport">` - Mobile responsiveness settings
- `<title>` - Page title shown in browser tabs
- `<meta name="description">` - Page description for search engines

### 4. The Body Section

The `<body>` contains all visible content on your web page.

## Best Practices

### 1. Use Semantic HTML

Choose elements based on meaning, not appearance. CSS handles styling.

```html
<!-- ‚ùå Bad -->
<div class="heading">Page Title</div>

<!-- ‚úÖ Good -->
<h1>Page Title</h1>
```

### 2. Write Valid HTML

Use proper nesting, close all tags, and include required attributes.

```html
<!-- ‚ùå Bad -->
<p>Paragraph <strong>bold text</p></strong>

<!-- ‚úÖ Good -->
<p>Paragraph <strong>bold text</strong></p>
```

### 3. Use Meaningful Alt Text

Describe what images show, not just "image" or "photo."

```html
<!-- ‚ùå Bad -->
<img src="dog.jpg" alt="image">

<!-- ‚úÖ Good -->
<img src="dog.jpg" alt="Golden retriever playing fetch in the park">
```

### 4. Structure with Headings

Use headings to create a logical document outline.

```html
<!-- ‚úÖ Good heading structure -->
<h1>Website Title</h1>
  <h2>Main Section</h2>
    <h3>Subsection</h3>
    <h3>Another Subsection</h3>
  <h2>Another Main Section</h2>
```

## Common Mistakes to Avoid

### 1. Missing DOCTYPE

Always include `<!DOCTYPE html>` at the top of your documents.

### 2. Unclosed Tags

HTML requires properly closed tags (except self-closing tags like `<img>` and `<br>`).

### 3. Missing Alt Attributes

Images should always have alt text for accessibility.

### 4. Improper Heading Hierarchy

Don't skip heading levels - use h1, then h2, then h3, etc.
