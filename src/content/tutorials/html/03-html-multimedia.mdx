### Essential Image Attributes

**Core attributes every image should have:**
- `src` - Path to the image file (required)
- `alt` - Alternative text for accessibility (required)
- `width` - Image width in pixels
- `height` - Image height in pixels
- `title` - Tooltip text on hover
- `loading` - Loading behavior (lazy, eager)

> **💡 Alt Text Best Practice** Alt text should describe what's important about the image in context. For decorative images, use empty alt text: `alt=""`.

#### Advanced Responsive Images Techniques

**1. Using `srcset` for Resolution Switching**
The `srcset` attribute allows browsers to choose the most appropriate image based on device capabilities:

```html
<img src="photo-medium.jpg"
     srcset="photo-small.jpg 480w,
             photo-medium.jpg 800w,
             photo-large.jpg 1200w,
             photo-xlarge.jpg 1600w"
     sizes="(max-width: 600px) 480px,
            (max-width: 1000px) 800px,
            1200px"
     alt="Responsive photo">
```

**2. Using `<picture>` for Art Direction**
The picture element gives you complete control over which image displays at different breakpoints:

```html
<picture>
  <!-- Mobile: Portrait crop -->
  <source media="(max-width: 600px)" 
          srcset="hero-mobile-portrait.jpg">
  
  <!-- Tablet: Landscape crop -->
  <source media="(max-width: 1200px)" 
          srcset="hero-tablet-landscape.jpg">
  
  <!-- Desktop: Wide crop -->
  <source media="(min-width: 1201px)" 
          srcset="hero-desktop-wide.jpg">
  
  <!-- Fallback -->
  <img src="hero-default.jpg" alt="Hero image">
</picture>
```

**3. Format Selection with `<picture>`**
Combine modern formats with fallbacks for optimal performance:

```html
<picture>
  <!-- Modern browsers: AVIF -->
  <source srcset="image.avif" type="image/avif">
  
  <!-- Widely supported: WebP -->
  <source srcset="image.webp" type="image/webp">
  
  <!-- Fallback: JPEG -->
  <img src="image.jpg" alt="Optimized image">
</picture>
```

**4. Density-Based Switching**
Handle high-DPI displays (Retina, etc.):

```html
<img src="photo.jpg"
     srcset="photo.jpg 1x,
             photo@2x.jpg 2x,
             photo@3x.jpg 3x"
     alt="Sharp image on all displays">
```

**Comprehensive Image Format Guide:**

#### JPEG (Joint Photographic Experts Group)
- **Best for**: Photographs, complex images with many colors
- **Compression**: Lossy compression (smaller files, some quality loss)
- **Features**: No transparency support, excellent for photos
- **File sizes**: Medium to small depending on quality settings
- **Browser support**: Universal (100%)
- **Use case**: Product photos, portraits, landscapes

#### PNG (Portable Network Graphics)
- **Best for**: Images with transparency, graphics with text, simple illustrations
- **Compression**: Lossless compression (larger files, perfect quality)
- **Features**: Alpha transparency support, 24-bit color depth
- **File sizes**: Larger than JPEG, smaller than uncompressed
- **Browser support**: Universal (100%)
- **Use case**: Logos with transparency, UI elements, screenshots

#### WebP (Web Picture Format)
- **Best for**: All types of images (modern replacement for JPEG/PNG)
- **Compression**: Both lossy and lossless options available
- **Features**: Transparency support, 25-50% smaller than JPEG/PNG
- **File sizes**: Significantly smaller than equivalent JPEG/PNG
- **Browser support**: 97%+ (IE not supported)
- **Use case**: All modern web images with fallback

#### AVIF (AV1 Image Format)
- **Best for**: Next-generation image compression
- **Compression**: Superior to WebP (up to 50% smaller than WebP)
- **Features**: HDR support, transparency, wide color gamut
- **File sizes**: Smallest of all formats
- **Browser support**: 85%+ (growing rapidly)
- **Use case**: High-performance websites, mobile optimization

#### SVG (Scalable Vector Graphics)
- **Best for**: Icons, logos, simple graphics, illustrations
- **Compression**: Vector-based (infinitely scalable)
- **Features**: CSS/JavaScript interactive, perfect at any size
- **File sizes**: Very small for simple graphics
- **Browser support**: Universal (100%)
- **Use case**: Icons, logos, simple illustrations

### Video Attributes

**Key video attributes:**
- `controls` - Show playback controls
- `autoplay` - Start playing automatically (requires muted)
- `muted` - Start with audio muted
- `loop` - Repeat the video
- `poster` - Thumbnail image shown before play
- `preload` - How much to preload (none, metadata, auto)
- `playsinline` - Play inline on mobile devices

> **⚠️ Autoplay Warning** Most browsers block autoplay with sound. Always use `muted` with `autoplay` and provide controls for users to unmute.

### Video Formats and Codecs Deep Dive

Understanding video formats is crucial for web optimization:

#### Container Formats vs. Codecs
- **Container**: The file format that holds video, audio, and metadata (MP4, WebM, AVI)
- **Codec**: The compression algorithm that encodes/decodes the actual video data (H.264, VP9, AV1)

#### Major Video Formats for Web

**MP4 (MPEG-4 Part 14)**
- **Codecs**: Usually H.264 (older) or H.265/HEVC (newer)
- **Browser support**: Universal (100%)
- **File sizes**: Good compression with H.264, excellent with H.265
- **Use case**: Primary format for maximum compatibility
- **Pros**: Universal support, good quality-to-size ratio
- **Cons**: H.264 is aging, larger files than modern alternatives

**WebM**
- **Codecs**: VP8 (older) or VP9 (recommended)
- **Browser support**: 98%+ (not supported in Safari on iOS < 14)
- **File sizes**: 25-50% smaller than MP4/H.264
- **Use case**: Modern web video with fallback to MP4
- **Pros**: Open source, excellent compression, royalty-free
- **Cons**: Limited Safari support on older iOS

**AV1 (.mp4 or .webm container)**
- **Codecs**: AV1 (cutting-edge)
- **Browser support**: 85%+ (Chrome, Firefox, Edge, Safari 17+)
- **File sizes**: 30-50% smaller than H.264, 20% smaller than VP9
- **Use case**: Future-proof video for high-performance sites
- **Pros**: Best compression available, royalty-free, HDR support
- **Cons**: Limited support, high encoding complexity

#### Optimal Video Format Strategy

```html
<video controls poster="video-poster.jpg">
  <!-- Future: AV1 for maximum compression -->
  <source src="video.av1.mp4" type="video/mp4; codecs=av01.0.08M.10">
  
  <!-- Modern: VP9 in WebM -->
  <source src="video.webm" type="video/webm; codecs=vp9">
  
  <!-- Fallback: H.264 in MP4 -->
  <source src="video.mp4" type="video/mp4; codecs=avc1.4d002a">
  
  <!-- Ultimate fallback message -->
  Your browser doesn't support modern video formats.
</video>
```

### Audio Formats Deep Dive

Understanding audio formats helps you choose the right one for your content:

#### Comprehensive Audio Format Guide

**MP3 (MPEG Layer 3)**
- **Compression**: Lossy (smaller files, some quality loss)
- **Browser support**: Universal (100%)
- **File sizes**: Good compression, widely optimized
- **Quality**: Good at 128kbps+, excellent at 192kbps+
- **Use case**: Primary format for maximum compatibility
- **Pros**: Universal support, mature ecosystem, predictable file sizes
- **Cons**: Patented codec, not the most efficient modern option

**AAC (Advanced Audio Coding)**
- **Compression**: Lossy (better than MP3 at same bitrates)
- **Browser support**: 98%+ (native in Safari, Chrome, supported in Firefox)
- **File sizes**: 25% smaller than MP3 at equivalent quality
- **Quality**: Superior to MP3, especially at lower bitrates
- **Use case**: High-quality audio, Apple ecosystem
- **Pros**: Better compression than MP3, supported by Apple
- **Cons**: Not as universal as MP3

**OGG Vorbis**
- **Compression**: Lossy (excellent quality-to-size ratio)
- **Browser support**: 85%+ (Chrome, Firefox, Edge; not Safari)
- **File sizes**: Similar or smaller than MP3 with better quality
- **Quality**: Superior to MP3 at equivalent bitrates
- **Use case**: Open-source projects, modern web audio
- **Pros**: Open source, royalty-free, excellent compression
- **Cons**: No Safari support

**FLAC (Free Lossless Audio Codec)**
- **Compression**: Lossless (perfect quality, larger files)
- **Browser support**: 85%+ (Chrome, Firefox, Edge; Safari 17+)
- **File sizes**: 50-60% of original WAV size
- **Quality**: Perfect reproduction of original
- **Use case**: High-fidelity audio, archival purposes
- **Pros**: Lossless quality, open source
- **Cons**: Large file sizes, limited mobile support

**WAV (Waveform Audio File)**
- **Compression**: Usually uncompressed (huge files)
- **Browser support**: Universal (100%)
- **File sizes**: Very large (10MB+ per minute)
- **Quality**: Perfect (if uncompressed)
- **Use case**: Short sound effects, development/testing
- **Pros**: Universal support, perfect quality
- **Cons**: Extremely large files

#### Optimal Audio Strategy

```html
<audio controls preload="metadata">
  <!-- High quality: FLAC for audiophiles -->
  <source src="audio.flac" type="audio/flac">
  
  <!-- Modern: OGG Vorbis for quality -->
  <source src="audio.ogg" type="audio/ogg; codecs=vorbis">
  
  <!-- Alternative: AAC for better compression -->
  <source src="audio.aac" type="audio/aac">
  
  <!-- Fallback: MP3 for universal support -->
  <source src="audio.mp3" type="audio/mpeg">
  
  Your browser doesn't support modern audio formats.
</audio>
```

#### Audio Quality Guidelines

**For Music & Podcasts:**
- MP3: 192kbps minimum, 320kbps for high quality
- AAC: 128kbps good, 192kbps excellent
- OGG: 160kbps good, 192kbps excellent

**For Voice/Speech:**
- MP3: 64kbps acceptable, 128kbps good
- AAC: 48kbps acceptable, 96kbps good
- OGG: 64kbps acceptable, 96kbps good

#### Complete Multimedia Accessibility Guide

Making multimedia accessible requires understanding different user needs:

**Visual Impairments:**
- **Total blindness**: Need alt text, audio descriptions, screen reader compatibility
- **Low vision**: Need high contrast, scalable text, zoom support
- **Color blindness**: Avoid color-only information conveyance

**Hearing Impairments:**
- **Deafness**: Need captions, sign language interpretation, visual alerts
- **Hard of hearing**: Need volume control, caption options, visual indicators

**Motor Impairments:**
- **Limited mobility**: Need keyboard navigation, large click targets, pause controls
- **Tremors**: Need stable controls, click tolerance, pause functionality

**Cognitive Impairments:**
- **Processing difficulties**: Need clear navigation, pause/replay options, simple controls
- **Attention disorders**: Need minimal distractions, clear focus indicators

## Performance Optimization Deep Dive

Modern websites must balance rich multimedia with fast loading times. Here's how to optimize effectively:

### 1. Advanced Image Optimization

#### Format Selection Strategy
```html
<!-- Progressive format fallback for maximum efficiency -->
<picture>
  <!-- Next-gen: AVIF (50% smaller than JPEG) -->
  <source srcset="hero.avif" type="image/avif">
  <!-- Modern: WebP (25-35% smaller than JPEG) -->
  <source srcset="hero.webp" type="image/webp">
  <!-- Fallback: Optimized JPEG -->
  <img src="hero.jpg" alt="Hero image" loading="lazy">
</picture>
```

#### Compression Guidelines
- **JPEG**: Use 75-85% quality for photos, 60-75% for thumbnails
- **WebP**: Use 75-90% quality for lossy, or lossless for transparency
- **AVIF**: Use 50-70% quality (equivalent to 85-95% JPEG)
- **PNG**: Use tools like pngquant for 8-bit optimization

#### Responsive Image Optimization
```html
<img src="photo-800.jpg"
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w,
             photo-1600.jpg 1600w"
     sizes="(max-width: 600px) 400px,
            (max-width: 1000px) 800px,
            (max-width: 1400px) 1200px,
            1600px"
     alt="Responsive optimized photo"
     loading="lazy">
```

#### Critical Images Strategy
```html
<!-- Above-fold hero images: preload -->
<link rel="preload" as="image" href="hero.webp">

<!-- Above-fold images: eager loading -->
<img src="logo.svg" alt="Company logo" loading="eager">

<!-- Below-fold images: lazy loading -->
<img src="content.jpg" alt="Content image" loading="lazy">
```

### 2. Video Performance Mastery

#### Multi-Bitrate Streaming
```html
<video controls poster="poster.webp" preload="metadata">
  <!-- 4K for high-end devices -->
  <source src="video-4k.av1.mp4" 
          type="video/mp4; codecs=av01.0.12M.10"
          media="(min-width: 2560px)">
  
  <!-- 1080p for desktop -->
  <source src="video-1080p.webm" 
          type="video/webm; codecs=vp9"
          media="(min-width: 1024px)">
  
  <!-- 720p for mobile -->
  <source src="video-720p.mp4" 
          type="video/mp4; codecs=avc1.4d002a">
</video>
```

#### Smart Preloading Strategy
```html
<!-- Critical video: preload metadata -->
<video preload="metadata" controls>
  <source src="intro.mp4" type="video/mp4">
</video>

<!-- Background video: preload none, autoplay muted -->
<video preload="none" autoplay muted loop playsinline>
  <source src="background.webm" type="video/webm">
</video>

<!-- Long-form content: preload none -->
<video preload="none" controls poster="thumbnail.webp">
  <source src="documentary.mp4" type="video/mp4">
</video>
```

#### Video Optimization Checklist
- ✅ Use modern codecs (AV1 > VP9 > H.264)
- ✅ Implement adaptive bitrate streaming for long videos
- ✅ Generate poster images in next-gen formats
- ✅ Use media queries for device-specific video sizes
- ✅ Implement video lazy loading for below-fold content

### 3. Audio Performance

#### Format and Compression Strategy
```html
<audio controls preload="metadata">
  <!-- High quality for music -->
  <source src="song.flac" type="audio/flac">
  <source src="song-320.mp3" type="audio/mpeg">
  
  <!-- Optimized for speech/podcasts -->
  <source src="podcast-96k.ogg" type="audio/ogg">
  <source src="podcast-128k.mp3" type="audio/mpeg">
</audio>
```

#### Quality vs. Size Guidelines
**Music/High Quality:**
- FLAC: Lossless (large files, perfect quality)
- MP3 320kbps: Near-transparent quality
- AAC 192kbps: Excellent quality, smaller files

**Speech/Podcasts:**
- MP3 96kbps: Good quality, small files
- AAC 64kbps: Acceptable quality, very small
- Opus 48kbps: Best compression for speech

## Multimedia Security Considerations

Security is crucial when handling user-uploaded content and external media:

### 1. Content Security Policy (CSP)

Protect against XSS attacks through multimedia:

```html
<!-- Meta tag CSP for multimedia -->
<meta http-equiv="Content-Security-Policy" 
      content="img-src 'self' https://trusted-cdn.com; 
               media-src 'self' https://trusted-media.com; 
               object-src 'none';">
```

**CSP Headers for multimedia:**
```
Content-Security-Policy: 
  img-src 'self' https://cdn.example.com data:;
  media-src 'self' https://video.example.com;
  frame-src 'self' https://youtube.com https://vimeo.com;
  object-src 'none';
```

### 2. File Upload Security

When accepting user uploads:

```html
<!-- Server-side validation required -->
<form enctype="multipart/form-data" method="POST">
  <input type="file" 
         accept="image/jpeg,image/png,image/webp"
         name="upload">
</form>
```

**Security checklist for uploads:**
- ✅ Validate file extensions AND MIME types
- ✅ Scan files for malware
- ✅ Limit file sizes (images: 5MB, videos: 100MB)
- ✅ Strip EXIF data from images
- ✅ Store uploads outside web root
- ✅ Use virus scanning APIs
- ✅ Implement rate limiting

### 3. Hotlinking Protection

Prevent unauthorized use of your media:

```html
<!-- Use referrer policy to control referer information -->
<img src="protected-image.jpg" 
     alt="Protected content"
     referrerpolicy="strict-origin-when-cross-origin">

<!-- Server configuration (Apache .htaccess) -->
<!-- 
RewriteEngine On
RewriteCond %{HTTP_REFERER} !^https?://(www\.)?yourdomain\.com [NC]
RewriteRule \.(jpg|jpeg|png|gif|webp|mp4|webm)$ - [F,L]
-->
```

### 4. Privacy Protection

Protect user privacy in multimedia:

```html
<!-- Remove location data from images -->
<img src="photo-without-exif.jpg" alt="Privacy-safe photo">

<!-- Use privacy-enhanced embeds -->
<iframe src="https://www.youtube-nocookie.com/embed/VIDEO_ID"
        title="Privacy-enhanced YouTube embed"
        loading="lazy">
</iframe>

<!-- Implement lazy loading for external content -->
<div class="external-embed" data-src="https://external-service.com/embed">
  <button onclick="loadEmbed(this)">
    Load External Content (Privacy Notice)
  </button>
</div>
```

### 5. Third-Party Content Risks

Safely embed external content:

```html
<!-- Sandbox external content -->
<iframe src="https://external-content.com"
        sandbox="allow-scripts allow-same-origin"
        referrerpolicy="strict-origin"
        loading="lazy">
</iframe>

<!-- Use SRI for external resources -->
<link rel="preload" 
      as="image" 
      href="https://cdn.example.com/image.webp"
      integrity="sha384-hash-of-the-image"
      crossorigin="anonymous">
```

### 6. HTTPS and Mixed Content

Ensure secure media delivery:

```html
<!-- Always use HTTPS for media -->
<img src="https://secure-cdn.com/image.jpg" alt="Secure image">

<!-- Avoid mixed content issues -->
<video controls>
  <source src="https://secure-video.com/video.mp4" type="video/mp4">
</video>

<!-- Use protocol-relative URLs cautiously -->
<!-- Better: Always specify https: -->
<img src="https://cdn.example.com/image.jpg" alt="Secure image">
```

## Common Multimedia Mistakes

### 1. Missing Alt Text
Always provide meaningful alt text for images.

### 2. Autoplay with Sound
Don't autoplay videos with sound - it's annoying and often blocked.

### 3. Huge File Sizes
Optimize multimedia files for web delivery.

### 4. No Fallbacks
Always provide fallback content for unsupported browsers.

### 5. Accessibility Oversights
Include captions, transcripts, and descriptions for multimedia content.
